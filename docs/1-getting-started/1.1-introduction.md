# 1.1 产品介绍

RPG-Agent-MV 是面向 RPG Maker MV 制作者的本地桌面工具。它将 AI Agent 接入现有 RMMV 工程，用于生成事件、批量修改既有事件，并辅助处理脚本、插件和项目状态问题。

产品目标是减少事件录入、重复修改和脚本排查中的机械工作，使制作者能够以自然语言描述需求，并将结果落到真实工程中。

## 核心能力

RPG-Agent-MV 当前重点支持以下工作：

- **事件生成**：根据自然语言需求生成 RMMV 事件内容和事件命令。新事件默认进入待放置队列，由用户在地图上确定位置。
- **既有事件修改**：对已有地图事件进行批量修改，适用于统一调整台词、条件、开关变量或同类事件逻辑。
- **脚本与插件辅助**：读取插件配置和项目状态，辅助编写或调整项目中的 JavaScript 脚本和插件相关改动。
- **项目状态读取**：读取地图、事件、素材、数据库、插件和开关变量等上下文，减少脱离工程事实的生成结果。

## 界面组成

RPG-Agent-MV 的主要界面由三个区域组成：

- **编辑器**：查看地图和事件，处理待放置事件，并管理暂存改动。
- **控制台**：查看资产库、项目管理、运行日志和设置。
- **Agent 面板**：提交自然语言需求、回答 ASK 确认、查看事件预览和任务输出。

## Agent 体系

Agent 的能力和行为由以下机制共同决定：

| 机制 | 作用 |
|------|------|
| **MCP 工具** | 读取项目事实，执行地图和事件相关操作 |
| **Skill** | 提供专项任务说明、流程和约束 |
| **Subagent** | 处理可拆分的辅助任务；当前产品以单执行者为默认模式 |
| **Rules** | 定义常驻运行纪律，包括写入边界、ASK、错误处理和安全约束 |

详见 [第 4 章 Agent 体系](../4-agent-system/4.1-mcp-tools.md)。

## 能力边界

RPG-Agent-MV 不承诺以下能力：

- 从零自动生成完整游戏。
- 完整复刻 RPG Maker MV 编辑器。
- 默认自动生成地图、美术或音频。
- 自动决定新事件在地图上的精确落点。
- 在缺少配置、素材、地图或上下文时继续伪造结果。

关键剧情取舍、地图选择、新事件放置、素材选择和高风险冲突处理仍由用户确认。
