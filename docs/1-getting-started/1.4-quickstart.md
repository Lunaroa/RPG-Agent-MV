# 1.4 快速上手

本节通过一个小型事件任务说明基础流程：提出需求、读取项目事实、处理关键决策、生成事件、人工放置、查看改动并决定是否保留。

## 开始前确认

- 当前项目为目标 RMMV 项目。
- 编辑器可以打开目标地图。
- 设置中存在可用模型。
- 当前工程没有大量未确认改动。

## 示例任务

首次任务建议选择简单提示牌或 NPC 提示。可在 Agent 面板输入：

```text
为当前城镇地图准备一个待放置的告示牌事件。玩家调查后显示一句话，提醒去旅馆找村长。
不要改主线；目标地图不明确时进入 ASK 决策。预览确认后由用户选择「应用」，再在桌面端地图画布拖放位置。
```

如果已知地图编号，可进一步明确目标地图：

```text
为 Map003 准备一个待放置的告示牌事件。玩家调查后显示：村长今晚在旅馆等你。
不要决定坐标；事件进入待放置队列后，由用户拖放到村口附近。
```

## 执行流程

1. **读取项目事实**：Agent 读取地图、事件、开关变量等上下文。
2. **处理关键决策**：遇到地图选择、素材选择或剧情方向不明确时，系统通过 ASK 暂停等待决策。
3. **生成与预览**：Agent 注册事件内容，界面显示事件预览。
4. **应用到队列**：在确认卡片中选择「应用」，事件进入待放置队列。
5. **人工放置**：在 RPG Agent MV 桌面端地图画布上拖放待放置事件，系统写入地图数据。
6. **查看改动**：在聊天流、事件预览、编辑器或运行日志中查看本次结果。
7. **保留或丢弃**：检查符合预期后保留改动；不符合预期时丢弃本轮改动或继续要求调整。

## 首次任务检查项

- 项目和地图是否识别正确。
- 不确定事项是否进入 ASK。
- 事件文本与需求是否一致。
- 预览确认后是否选择「应用」进入待放置队列。
- 事件放置位置是否由用户在桌面端地图画布决定。
- 是否存在阻断性错误或风险提示。
