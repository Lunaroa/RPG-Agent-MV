# 1.5 设置

本节说明模型供应商、界面设置、工具权限和密钥边界。

## 模型配置

进入控制台的设置页，在模型区域确认以下项目：

- 已配置供应商。
- API Key 正确。
- 默认模型可用。
- 运行配置已保存并同步。

如果界面提示尚无 API 供应商，应先导入供应商预设或添加自定义供应商，再填写 API Key。

## 供应商配置

一个供应商通常包含名称、Base URL、API Key 和默认模型。

填写 API Key 后可尝试拉取模型列表。拉取失败时，应依次检查 API Key、Base URL 和供应商是否支持模型列表接口。详见 [5.3 模型与运行检测](../5-faq/5.3-model-check.md)。

## 界面设置

界面设置仅影响显示效果，例如聊天字号和聊天面板宽度。此类设置不改变 Agent 的判断能力或工具权限。

## 项目版本管理（控制台）

**版本管理**（可选）用于事件身份登记、剧情页同步与 Git 基线。地图与事件编辑默认可直接使用，无需先启用。该开关在 **控制台** 首页右上角（「启用版本管理」或「仅启用编辑」），不在本「设置」页。详见 [2.1 接入与切换项目](../2-projects/2.1-project.md#受控编辑与-git-基线)。

## 权限与规则

设置页可查看运行策略、工具策略、规则与技能。

- 工具策略决定 Agent 可调用的工具范围。
- 运行中的会话通常不会立即继承工具策略改动。
- 产品规则由发布版维护，用户无需修改。
- 规则和技能决定 Agent 处理 RMMV 工程的边界和流程。详见 [4.4 Rules](../4-agent-system/4.4-rules.md) 和 [4.2 Skill](../4-agent-system/4.2-skill.md)。

## 个人偏好

除模型和工具权限外，可在 `RPG-Agent-MV/config/opencode/instructions/personal-preferences.md` 中填写个人偏好，例如汇报风格、对话口吻或命名习惯。保存后，下次启动 Agent 会话时生效。详见 [4.4 Rules](../4-agent-system/4.4-rules.md)。

## 密钥安全

API Key 和本地凭证只应保存在本地配置中。不要将真实密钥写入聊天、文档、截图或仓库。
