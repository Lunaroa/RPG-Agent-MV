# 2.1 接入与切换项目

本节说明 RPG Maker MV/MZ 工程的接入要求、推荐项目状态和多项目切换注意事项。

## 项目要求

RPG Agent MV 需要连接真实、完整的 RPG Maker MV 工程，或 MZ 1.10.0 编辑器源工程。目标项目至少应满足以下条件：

- 可由对应的 RPG Maker 编辑器正常打开。
- MZ 工程包含 `game.rmmzproject`，核心脚本版本严格为 1.10.0，且没有部署布局或加密资源。
- 存在地图数据和数据库数据。
- 任务所需素材真实存在。
- 项目路径可由 RPG Agent MV 读取。

项目不完整或路径不可访问时，Agent 应停止执行并说明原因。

## 接入步骤

1. 使用对应的 RPG Maker 编辑器打开目标项目并保存；旧版 MZ 工程需先在 MZ 1.10.0 中执行“更新核心脚本”。
2. 启动 RPG Agent MV。
3. 在控制台确认当前项目。
4. 在编辑器打开地图树。
5. 选择一张地图，确认画布可正常显示。
6. 确认项目状态无误后再发起 Agent 任务。

项目无法接入时，参见 [5.2 项目识别失败](../5-faq/5.2-project-detection.md)。

## 推荐项目状态

以下状态有助于 Agent 正确读取上下文：

- 地图命名清晰。
- 关键开关和变量已命名。
- 角色、地点和任务线名称稳定。
- 当前工程没有大量未确认改动。
- 项目位于 Git 工作树中。

命名越稳定，Agent 越容易识别目标对象和上下文。

## 切换项目

在控制台切换项目后，应确认顶部项目名旁的 MV/MZ 标识正确。切换后如出现数据读取失败，应先在对应编辑器中保存项目，再回到 RPG Agent MV 刷新。引擎由系统自动识别，不能手动切换。

## MZ 试玩运行时

使用 MZ 工程不要求安装 RPG Maker MZ。编辑能力只依赖源工程；普通试玩与隔离战斗测试会静态校验工程根目录的 `Game.exe`、`nw.dll`、`icudtl.dat`、语言包等 NW.js 文件。识别工程时不会执行 `Game.exe`。隔离测试继续使用系统临时目录，但运行器保留在源工程中，只接收临时工程路径，不会被复制。

缺少完整本地运行器时，地图、事件、数据库、素材、插件和暂存仍可使用；普通试玩与战斗测试会明确阻止启动并指出缺少的文件。

## 游戏级规则（可选）

若某个工程需要长期生效的专用说明（例如世界观口吻、命名表、禁止改动的主线范围），可在游戏工程目录中放置 `AGENTS.md`。Agent 会将其作为该项目的补充规则。详见 [4.4 Rules](../4-agent-system/4.4-rules.md)。

## 受控编辑与版本管理

地图与事件编辑**默认即可使用**，无需先启用版本管理。改动会进入暂存，可在 [2.3 暂存、应用与回退](2.3-staging.md) 中审阅后写回工程。

**版本管理**（控制台首页右上角）是可选增强能力：启用后会登记事件身份、同步剧情页指纹，并保存一份本地版本快照。启用时可填写版本说明；之后也可随时点击 **保存版本** 记入本地历史。**不需要远程仓库或联网同步。**

启用方式在**控制台首页**（不是「设置」页）：

1. 打开左侧导航的 **控制台**。
2. 点击 **启用版本管理**（会先保存当前项目快照；失败则不启用）。
3. 可选填写 **版本说明**（例如「项目初稿」）。
4. 若暂时不保存快照，可选 **仅启用编辑**（只打开事件身份登记）。

Agent 在修改地图或事件前应先读取项目事实。写入后，用户可以通过暂存对比、运行日志查看改动范围；启用版本管理后还可通过 **保存版本** 留下带说明的本地记录。
