# 2.2 地图、事件与资产

本节说明编辑器中的地图视图、事件模式、待放置事件和资产数据库。

## 地图树与工作模式

编辑器左侧为地图树。选择地图后，画布显示对应地图。若没有可加载地图，应先确认项目是否为有效 RMMV 工程。

画布包含两种工作模式：

- **地图模式**：查看地图，并使用当前已接入的受控图块、阴影和区域编辑能力。该模式不是 RPG Maker MV 原生地图编辑器的完整替代，复杂自动图块和通行旗标仍需在 RMMV 中确认。
- **事件模式**：处理事件的新建、编辑、移动、复制、粘贴、删除，以及放置 Agent 生成的待放置事件。

仅进行事件任务时，应避免在地图模式中产生无关图块改动。

## 待放置事件

Agent 可以设计并注册新事件，但不决定新事件的最终地图坐标。流程分三步：

1. **注册**：`registry.register` 写满 `implementation.pages[].commands[]` 后，事件进入右侧栏**预览**（`reviewing`），尚未进入待放置队列。
2. **审批**：Agent **直接**用 `question` 询问「应用到待放置队列？」（选项：应用 / 调整 / 取消）；用户选「应用」后，UI 才把事件批准进待放置队列。
3. **拖放**：用户在 **RPG-Agent-MV 桌面端地图画布**（不是外部 RPG Maker MV 编辑器）将待放置事件拖到目标格；系统调用 `createPlacementEvent` 写入 Map JSON。改动会进入暂存，可在 [2.3 暂存、应用与回退](2.3-staging.md) 中审阅后写回工程。

新建、修改、删除、复制已有地图事件同样**无需**先启用版本管理；若需事件身份登记与 Git 基线，可到 **控制台** 首页启用「版本管理」或「仅启用编辑」。详见 [2.1 接入与切换项目](2.1-project.md#受控编辑与-git-基线)。

Agent **不得**引导用户去外部 RMMV 编辑器选坐标，也**不得**用 `editor.update` / `editor.move` 替新事件写入 x/y。

## 事件检查项

查看或修改事件时，建议确认以下内容：

- 事件位置。
- 触发方式。
- 事件页条件。
- 对话、分支和演出。
- 开关、变量和素材引用。

## 资产与数据库

控制台的资产库用于查看地图、技能、图块、角色、图片和音频，并识别明显引用问题。项目管理入口用于查看开关变量、公共事件、音频、插件和数据库。

Agent 写入事件时引用的素材必须真实存在。缺失素材应作为阻断问题处理，不应由 Agent 虚构替代资源。
