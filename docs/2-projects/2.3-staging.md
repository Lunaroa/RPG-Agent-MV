# 2.3 暂存、应用与回退

本节说明 RPG-Agent-MV-MV 的暂存机制，以及如何查看、应用或丢弃本轮改动。

## 暂存机制

Agent 修改地图或事件时，会在写入前创建源文件快照。写入完成后，用户可以查看改动，并决定保留或丢弃。

暂存可能覆盖地图、事件和项目级改动。它用于降低误写入风险，但不替代用户对改动范围的确认。

## 查看本次改动

应用前建议检查以下信息：

- Agent 在聊天流中的改动说明。
- 编辑器中地图或事件的实际变化。
- 项目位于 Git 工作树时的 `git diff`。
- 运行日志中的阻断错误或风险提示。

## 应用改动

满足以下条件后再应用改动：

- 改动范围符合任务目标。
- 地图和事件位置正确。
- 没有阻断性错误。
- 没有无关文件或无关事件改动。

## 丢弃改动

以下情况应丢弃本轮改动：

- 写入了错误地图或错误事件。
- 事件内容方向明显不符合需求。
- 写入范围越界。
- 存在未解决的阻断问题。
- 用户决定不保留本轮尝试。

丢弃后，源文件恢复到本轮写入前的快照状态。

## 暂存与保存版本

| 方式 | 作用 | 适用场景 |
|------|------|----------|
| **丢弃暂存** | 将本轮未保留改动还原到快照 | 当前任务结果不符合预期 |
| **保存版本** | 把已写入工程的改动记入本地版本历史（可写版本说明） | 需要留下带说明的里程碑，便于日后对比或回退 |

保存版本在控制台右上角 **版本管理** 区域操作，只需本机安装 Git，**不需要远程仓库**。启用版本管理后即可随时保存；若当前没有新改动，会提示「没有新的改动需要保存」。

## 冲突处理

应用暂存时如出现冲突，通常说明工程文件在暂存之后被外部工具修改。此时应停止应用，确认 RPG Maker MV 或其他工具是否改写了项目文件，再决定重新读取、丢弃暂存或人工处理冲突。
