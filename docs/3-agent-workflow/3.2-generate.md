# 3.2 从生成到落地

本节说明新事件从生成到进入 RMMV 工程的标准流程。

## 流程概览

1. **设计并注册**：Agent 设计事件内容，包括对话、分支、状态变化和必要演出，并将其注册为事件。注册时应包含完整事件命令，确保预览内容可检查。此时事件处于**预览**状态，尚未进入待放置队列。
2. **预览与决策**：界面显示事件预览。Agent **直接**用 `question` / `AskUserQuestion` 询问「应用到待放置队列？」（不问「要不要走流程」）。用户可选：
   - **应用**：批准进入待放置队列；
   - **调整**：按反馈修改后重新预览；
   - **取消**：放弃本轮事件。
3. **人工放置**：用户在 **RPG-Agent-MV 桌面端地图画布**（不是外部 RPG Maker MV 编辑器）将待放置事件拖到目标格。放置坐标由用户决定，Agent 不替代用户判断落点。

## 项目事实读取

Agent 修改地图或事件前，应先读取当前项目的事件、开关、变量等事实。这样可以降低新事件与既有状态、事件页条件或素材引用冲突的概率。

## 落地后的检查

事件放置后，回到 [2.3 暂存、应用与回退](../2-projects/2.3-staging.md) 查看本轮改动。结构检查只能说明机械结构未被拦截；剧情语气、演出节奏和任务目标仍需人工审阅。
