# 3.3 会话与日志

本节说明会话延续、运行日志和问题定位方式。

## 会话

Agent 面板保留每轮对话。用户可以基于上一轮结果继续提出修改要求，而不必重复完整上下文。运行过程中如发现方向错误，可以停止当前任务。

## 运行日志

控制台的运行日志记录 Agent 执行过程、项目检查结果和引擎输出。它主要用于定位以下问题：

- Agent 声称已修改，但工程中未看到结果。
- 任务报错或项目检查失败。
- 需要检查 Agent 是否读取了正确项目事实。

## 检查建议

不要将“任务运行结束”直接等同于“结果正确”。对于未触发 ASK、任务结果偏离预期或涉及写入的情况，应结合事件预览、编辑器实际变化和运行日志进行检查。
