# 4.1 MCP 工具

## 定义

MCP 工具是 Agent 访问外部能力的接口。RPG Agent MV 通过 MCP 工具读取 RMMV 项目事实，并执行地图、事件等受控操作。工具调用记录会出现在控制台运行日志中。

## 内置 RMMV 工具

RPG Agent MV 内置一组 RMMV 专用 MCP 工具，覆盖读取上下文、地图操作和事件操作。

| 工具 | 作用 |
|------|------|
| **读上下文**（`RmmvReadContext`） | 读取事件上下文、地图索引、素材/插件清点、完整数据库分页目录和单条全部嵌套数据；优先返回暂存版本 |
| **数据库准备**（`RmmvDatabase`） | 主 Agent 执行批量校验、预演、暂存和放弃；预演不写草稿，暂存生成受所有权保护的操作 |
| **数据库应用**（`RmmvDatabaseApply`） | 仅按操作编号应用已暂存数据库变更；调用前必须经过原生 ASK 审批和再次预检 |
| **地图**（`RmmvMap`） | 执行地图创建、属性修改、复制、删除、绘制，以及暂存应用或丢弃 |
| **事件**（`RmmvEvent`） | 新事件：`registry.register` → 右侧预览 → 用户「应用到待放置队列」→ 桌面端地图拖放（`createPlacementEvent`）。已有事件：补丁、反馈、`editor.update`/`editor.move`（**禁止**用于新事件落点或引导用户去外部 RMMV 编辑器） |
| **隔离验证**（`RmmvVerify`） | 主 Agent 在临时工程副本中覆盖全部暂存并运行有限探针；证据不完整时不会返回已验证 |

这些工具是 Agent 在 RMMV 工程中工作的核心能力。读取型子 Agent 只能读取，不能准备、暂存、应用数据库操作或启动隔离验证。

## 工作流工具

除 RMMV 工具外，主 Agent 还可调用**工作流提议**工具（`workflow.propose`）提交一份编排脚本，扇出多个只读子 Agent 并行或流水线执行（详见 [3.4 动态工作流](../3-agent-workflow/3.4-dynamic-workflow.md)）。该工具仅对非只读 Agent 开放；脚本提交后需用户在桌面端审批卡片上批准才会执行。

## 调用场景

- 写入事件前，Agent 调用读上下文工具获取项目事实。
- 涉及地图修改或事件落地时，Agent 调用地图和事件工具。
- 调用结果写入运行日志，供用户核对任务执行过程。

## 扩展 MCP 工具

如需接入产品内置能力之外的 MCP 服务器，需在高级配置中注册目标服务器、授权对应工具，并通过简单任务确认可用性。外部 MCP 属于扩展能力，其稳定性取决于对应服务器本身。
