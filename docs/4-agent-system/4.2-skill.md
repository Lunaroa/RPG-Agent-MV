# 4.2 Skill

## 定义

Skill 是 Agent 的专项能力说明。它将特定领域的流程、约束、参考资料和辅助脚本组织为一个目录，使 Agent 在相关任务中获得更明确的操作指导。

一个 Skill 目录通常包含：

- `SKILL.md`：核心说明文件。
- frontmatter：包含 `name` 和 `description`，用于标识和触发。
- 可选资源：脚本、参考文档、模板或示例。

## 内置 Skill

发布版默认包含 `skill-creator`。该 Skill 用于指导 Agent 创建或更新其他 Skill，减少手工编写格式的成本。

## 添加、修改和删除

Skill 放在产品根目录的 `RPG-Agent-MV/config/opencode/skills/` 下，无需额外注册。

- **添加**：在 `config/opencode/skills/` 下创建小写连字符命名的目录，例如 `my-skill/`，并放入包含 `name`、`description` 和正文的 `SKILL.md`。
- **修改**：直接编辑目标 Skill 的 `SKILL.md`。
- **删除**：删除对应 Skill 目录。

建议使用 `skill-creator` 创建或更新 Skill，以确保结构符合约定。

## 注意事项

- `description` 决定 Skill 何时被触发，应准确描述适用场景。
- `config/opencode/skills/` 是正式 Skill 区，只应放置成熟 Skill。
- 草稿和实验材料应放在运行时输出目录，不应直接进入正式 Skill 区。
