# 4.3 Subagent

## 定义

Subagent 是主 Agent 为完成辅助任务而临时启动的子任务能力。它适用于并行读取上下文、检索多处资料或执行可拆分的辅助分析。

该能力由系统内置提供，用户不需要手工创建 agent 文件。

## 默认模式

RPG-Agent-MV 默认允许主 Agent 派遣 subagent。`Agent` 工具位于默认执行者的工具白名单中，主 Agent 可在适合拆分的任务中调用该能力。

当前产品没有固定多角色流水线。默认工作方式仍是单执行者主导，子 agent 仅作为临时辅助。地图选图、新事件放置和导入仍为人工步骤。

## 默认执行者配置

默认执行者配置位于 `RPG-Agent-MV/config/agents/`。该配置决定 Agent 的工具权限和写入范围。

- **工具白名单（allow）**：允许使用的基础读写工具、计划模式、subagent 工具和 RMMV MCP 工具。
- **禁用项（deny）**：明确禁止的危险操作，例如读取密钥文件、`git reset --hard`、`git clean` 和递归删除。
- **权限**：定义是否允许写入源工程，以及写入是否需要人工批准。

默认执行者可以修改 RMMV 工程并调用辅助子任务，但禁止破坏性操作。需要收紧或放开某项能力时，应修改默认执行者配置。

## 注意事项

- 子 agent 类型由系统内置提供，不需要通过文件登记。
- 产品当前不提供固定多角色编排。
- 更复杂的协同模式属于后续扩展方向。
