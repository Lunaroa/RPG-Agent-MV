# 4.4 Rules

## 定义

Rules 是 Agent 的常驻运行规则。它定义工具使用、确认卡片（ASK）、写入边界和 RMMV 制作纪律等基础约束。

## 产品规则（只读）

RPG-Agent-MV 自带一套产品级运行规则，由发布版维护，覆盖角色边界、工具策略、RMMV 写入纪律、产物落盘和 Skill 沉淀等。

**用户无需也不应修改产品规则。** 规则在启动时自动加载，后续任务会始终遵循。

主要纪律包括：

- 仅使用当前运行时真实可用的工具；能力不可用时停止并说明原因。
- 写入地图或事件前先读取项目事实。
- **新事件**：先注册并预览 → 用户选「应用」进入待放置队列 → 在桌面端地图画布拖放（不由 Agent 决定坐标，不使用外部 RPG Maker MV 编辑器放置）。
- 缺少上下文或工具不覆盖时直接失败，不猜测补全。

## 个人偏好

你可以在个人偏好文件中补充通用规则之外的口味和习惯，例如：

- 汇报时先给结论，再给细节。
- 事件对话尽量短句，少用感叹号。
- 角色称呼或命名偏好。

**编辑位置**：`RPG-Agent-MV/config/opencode/instructions/personal-preferences.md`

保存后，下次启动 Agent 会话时会自动生效。个人偏好不能覆盖产品规则中的安全纪律和写入边界。

## 游戏级规则（可选）

如果某个 RMMV 游戏工程需要长期生效的专用说明（例如世界观口吻、命名表、禁止改动的主线范围），可在**游戏工程目录**中放置 `AGENTS.md`。

Agent 会读取距离当前项目最近的一份 `AGENTS.md`，作为该游戏的补充规则。没有则不加载。

## 维护原则

- 产品规则由发行版更新，用户通过升级产品获得新纪律。
- 个人偏好只描述习惯和口味，不要复制或改写产品规则正文。
- 游戏级 `AGENTS.md` 应聚焦该项目的长期约束，不要放密钥或无关路径说明。
- 修改偏好后，可用一个简单任务确认 Agent 行为符合预期。
