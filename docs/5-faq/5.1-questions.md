# 5.1 FAQ

## 是否可以自动生成完整游戏？

不可以。RPG Agent MV 面向已有 RMMV 工程，主要用于生成事件、批量修改既有事件，以及辅助处理脚本和插件问题。

## 是否会自动生成地图、美术和音频？

默认不会。RPG Agent MV 基于工程中已有地图和素材工作。缺少资源时，应补齐资源后再继续任务。

## 为什么需要 ASK？

地图选择、新事件放置、剧情取舍、素材选择和冲突处理均属于高风险决策。Agent 通过 ASK 请求用户确认，以避免自动越权。

ASK 是协作流程中的确认机制，但不是所有后端写入 API 的硬性锁。涉及写入的任务仍应检查改动范围。

## 项目检查通过后是否还需要人工确认？

需要。结构检查或运行检测只能说明机械条件满足要求。剧情语气、演出节奏、任务目标和地图落点仍需要人工确认。

## 结果不符合预期时如何处理？

如果方向基本正确，可以继续要求 Agent 调整。如果方向错误或写入范围不符合预期，应丢弃本轮改动或重新提交更明确的需求。

## 写入了错误地图如何处理？

不要保留本轮改动。应丢弃暂存，重新指定目标地图；如果是新事件，则重新从待放置队列拖放到正确位置。详见 [2.3 暂存、应用与回退](../2-projects/2.3-staging.md)。

## 模型配置失败如何处理？

进入设置检查供应商、API Key、Base URL 和默认模型。修复后先用简单任务确认模型可用。详见 [5.3 模型与运行检测](5.3-model-check.md)。

## 如何调整 Agent 行为？

Agent 的能力和纪律由 MCP 工具、Skill、Subagent 和 Rules 共同决定。详见 [第 4 章 Agent 体系](../4-agent-system/4.1-mcp-tools.md)。
