# 5.2 项目识别失败

本节说明项目无法识别或项目数据无法读取时的排查方式。

## 常见现象

- 项目列表为空，或当前项目名称不正确。
- 控制台显示项目数据读取失败。
- 编辑器显示当前工程没有可加载地图。
- Agent 提示无法读取项目上下文。

## 常见原因

- 选择目录不是 RPG Maker MV 工程。
- 项目数据文件缺失或损坏。
- 地图文件缺失。
- RPG Maker MV 中的项目尚未保存。
- 目录权限不足或文件被外部程序占用。

## 排查步骤

1. 使用 RPG Maker MV 打开目标项目。
2. 确认项目中存在地图。
3. 在 RPG Maker MV 中保存项目。
4. 回到 RPG Agent MV 重新选择或刷新项目。
5. 在控制台项目管理中确认地图、事件、数据库和音频是否可读取。

## 仍无法识别时

可将现象、项目路径、RMMV 打开结果和界面错误提示整理给 Agent，并明确要求只读检查，不执行写入操作。
