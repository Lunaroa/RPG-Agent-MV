# 5.2 项目识别失败

本节说明项目无法识别或项目数据无法读取时的排查方式。

## 常见现象

- 项目列表为空，或当前项目名称不正确。
- 控制台显示项目数据读取失败。
- 编辑器显示当前工程没有可加载地图。
- Agent 提示无法读取项目上下文。

## 常见原因

- 选择目录不是 RPG Maker MV 或受支持的 MZ 源工程。
- MZ 核心脚本无法报告可识别版本，或工程混入 MV 文件；`game.rmmzproject` 不是识别要求。
- 数据布局互相冲突，或缺少必要数据和核心文件。
- 可识别的旧版核心和加密资源会显示兼容警告，但不会仅因此拒绝导入；加密素材相关能力可能受限。
- 项目数据文件缺失或损坏。
- 地图文件缺失。
- 项目尚未在对应的 RPG Maker 编辑器中保存。
- 目录权限不足或文件被外部程序占用。

## 排查步骤

1. 使用对应的 RPG Maker 编辑器打开目标项目。
2. 确认项目中存在地图。
3. 保存项目；MZ 旧核心可在确认警告后继续使用，也可先备份，再在 MZ 1.10.0 中执行“更新核心脚本”以回到完整验证基线。
4. 回到 RPG Agent MV 重新选择或刷新项目。
5. 在控制台项目管理中确认地图、事件、数据库和音频是否可读取。

## 仍无法识别时

可将现象、项目路径、对应编辑器打开结果和界面错误提示整理给 Agent，并明确要求只读检查，不执行写入操作。系统不会通过执行工程脚本来猜测版本，也不会自动替换核心脚本。
