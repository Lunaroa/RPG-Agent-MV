# 5.3 模型与运行检测

本节说明模型不可用、运行失败以及本地状态冲突时的排查流程。

## 检查模型和供应商

设置中应存在已配置供应商和默认模型。供应商至少需要包含：

- 名称或 ID。
- 正确的 Base URL。
- 有效 API Key。
- 可用默认模型。
- 已保存并同步的运行配置。

## 执行运行检测

如果设置页提供运行检测，应先执行检测。检测失败时，不建议继续复杂任务。

常见失败原因包括：

- 没有可用模型。
- API Key 无效。
- Base URL 不正确。
- 当前运行入口被配置阻断。
- 模型不支持当前任务需要的能力。

## 修复顺序

1. 配置供应商。
2. 拉取模型列表。
3. 选择默认模型。
4. 保存并同步设置。
5. 使用简单任务确认模型可用。

## 环境与本地状态冲突

部分失败与模型无关，而是由本地状态导致：

- 环境变量指向错误项目。
- 供应商配置未保存或未同步到执行器。
- RPG Maker MV 正在占用或改写项目文件。
- 当前工作树存在未确认改动。

建议按以下顺序排查：确认当前项目路径，确认供应商和模型配置，在 RPG Maker MV 中保存项目，检查 Git 状态或编辑器暂存，再要求 Agent 重新读取项目事实。

不要通过临时兼容路径、复制工程或绕过配置错误来继续任务。路径或配置不明确时，应先完成确认。
