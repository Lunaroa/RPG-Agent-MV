# Requests And Confirmation

[Back to User Guide](../README.md)

Write production requests in terms of the game outcome you want, the target area, and any constraints the Agent must respect.

Good requests name the map, event, character, switch, variable, plugin, or database area when known. If the target is unclear, the Agent should ask before writing.

ASK cards are for decisions that block safe progress: selecting a target, approving a plan, choosing between mutually exclusive options, or confirming a risky write.

Do not rely on vague location instructions for new events. A request such as "add an NPC at the village entrance" should lead to a pending event and a user placement step, not an automatic coordinate guess.
